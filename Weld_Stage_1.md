## Route selection heuristic

- If the codebase is fragile, start with the safe route and add dynamic contracts behind feature flags\.
- If the page already has clean component boundaries, use the target route and wire state immediately\.
- If the goal is a screenshot\-worthy launch, use the cinematic route after the target route passes accessibility and performance gates\.
- If mobile or performance is currently weak, choose the constraint\-reduced route before adding ambient motion or extra motifs\.

# 08\. Accessibility, Performance, and Brand\-Safety Gates

## Accessibility gates

- Every foreground/background token pair used for text must pass contrast requirements for its text size\.
- Color cannot be the only indicator of selected, verified, error, available, or warning state\.
- Focus\-visible must use Studio Blue and remain obvious on all dark surfaces\.
- Reduced\-motion mode must replace motion feedback with color, border, label, and Output log feedback\.
- Do not use ultra\-low opacity text for important copy even if it looks cool\.

## Performance gates

- Token changes must not trigger layout shifts\.
- Avoid heavy backdrop filters on every panel; prefer flat depth and 1px borders\.
- Do not overuse glowing shadows; they are expensive and reduce visual hierarchy\.
- Use color transitions sparingly and keep them under 200ms unless motion stage says otherwise\.

## Brand\-safety gates

- Use Studio\-inspired colors and panel logic; do not imply official Roblox endorsement\.
- Badge wording should be proof\-specific: Proof verified, Roblox proof checked, or Game data verified\.
- Avoid official\-looking marks unless the legal/brand decision is already approved\.
